# 要件定義

## 1. プロダクト概要

このプロダクトは、日本時間でW杯を見る国内ユーザー向けのスマホWebアプリです。

単なる試合日程表ではなく、日本代表とユーザーが選んだ推し国に関係ある試合だけを、見る価値・突破条件・3位通過ライン・トーナメント分岐つきで提示します。

## 2. 基本方針

- 情報を増やすのではなく、見るべき情報だけに減らす
- 日本国内ユーザー向けに日本時間で最適化する
- 日本代表だけでなく、推し国を複数選べるようにする
- 順位計算や突破条件判定はAIに任せず、TypeScriptロジックで処理する
- AIは解説文、見る理由、X投稿文、UI文言生成に使う
- 初期MVPでは低コスト・静的Webアプリ運用を優先する

## 3. Must：初期MVPで必ず作るもの

- スマホ最適化Webアプリ
- 日本時間の全試合スケジュール
- 日本代表＋推し国設定
- 今日見るべき試合ランキング
- グループ順位表
- 3位通過ライン
- 試合詳細ページ
- 試合ごとの見る理由
- X共有カード生成
- ICSカレンダー出力
- 無料APIによる試合結果取得の基本実装
- GitHub Pagesでの公開
- GitHub Actionsによる結果更新の検証

## 4. Should：できれば初期に入れたいもの

- トーナメント分岐表示
- 推し国別ページ
- 朝まとめカード
- API不調時の手動補正
- データ検証スクリプト
- 重要度ロジックの調整
- X投稿文コピー機能

## 5. Could：余裕があれば追加するもの

- 過去対戦・因縁データ
- Grok連携用プロンプト集
- note記事化テンプレート
- PWA化
- 多言語対応
- 推し国別の共有URL
- 観戦メモ機能

## 6. Won't：初期では作らないもの

- ネイティブアプリ
- LINE通知
- ログイン機能
- 課金機能
- リアルタイム速報
- 試合中ライブ更新
- 勝敗予想メイン機能
- コミュニティ機能
- コメント欄
- 自動X投稿
- 公式大会ロゴ・公式素材を使ったデザイン

## 7. 対象ユーザー

主な対象は、日本国内でW杯を観戦するライト〜中級ファンです。

特徴は以下です。

- 日本代表は見る
- 海外の強豪国や推し国も2〜3チーム気になる
- 全104試合を追う余裕はない
- 深夜・早朝の試合を見るべきか判断したい
- グループ突破条件や3位通過ラインは詳しくない
- SNSや職場・友人との会話には乗りたい
- 速報アプリではなく、試合の意味の整理が欲しい

## 8. 主要機能

### 8.1 今日見るべき試合ランキング

ユーザーが今日見るべき試合を一目で判断できるようにします。

表示項目：

- 今日の日付
- 今日の試合一覧
- 重要度ランク S/A/B/C
- 日本時間のキックオフ時刻
- 対戦カード
- グループ/ステージ
- 日本または推し国への関係
- 見る理由の一言
- 寝不足価値

重要度：

- S：寝不足してでも見る価値あり
- A：結果は必ず追うべき
- B：ハイライトで十分
- C：寝ていい

### 8.2 推し国設定

ユーザーごとに関心のある国を設定できるようにします。

仕様：

- 日本代表を初期設定
- メイン推し国：1チーム
- サブ推し国：最大3チーム
- ゆる追い国：最大5チーム
- 設定はlocalStorageに保存
- ログインは不要

### 8.3 全試合スケジュール

表示項目：

- 試合ID
- 日時 日本時間
- 対戦国
- グループ/ステージ
- 会場
- ステータス
- スコア
- 重要度
- 関係する推し国

フィルター：

- 今日
- 明日
- 日本戦
- 推し国
- 重要度S/A
- グループ別
- 決勝トーナメント

### 8.4 グループ順位表

表示項目：

- グループ名
- チーム名
- 試合数
- 勝
- 分
- 敗
- 勝点
- 得点
- 失点
- 得失点差
- 暫定順位
- 状態ラベル

MVPでは、状態ラベルは以下から始めます。

- 確定
- 通過圏
- 圏外
- 未確定

### 8.5 3位通過ライン

12グループの3位チームを横断比較し、通過圏とボーダーを表示します。

表示項目：

- 3位チームランキング
- チーム名
- グループ
- 勝点
- 得失点差
- 総得点
- 試合数
- 通過圏/圏外/未確定
- 推し国との関係

重要機能：

- 上位8チームを通過圏として表示
- 9位以下を圏外として表示
- 推し国が3位の場合に強調
- 推し国が3位になり得る場合に注意表示

### 8.6 トーナメント分岐表

推し国がこのまま進んだ場合に、どこと当たりそうかを表示します。

MVPでは、複雑すぎる分岐は「未確定」と明示します。

表示例：

- 日本が1位通過した場合：想定相手A
- 日本が2位通過した場合：想定相手B
- 日本が3位通過した場合：組み合わせ条件次第

### 8.7 試合詳細ページ

表示項目：

- 対戦カード
- キックオフ時刻 日本時間
- グループ/ステージ
- 現在順位
- 重要度
- 見る理由
- 勝った場合の意味
- 引き分けの場合の意味
- 負けた場合の意味
- 推し国への影響
- 過去対戦・因縁
- X共有カード生成
- ICS追加

### 8.8 ICSカレンダー生成

LINE通知を初期対象外にする代わりに、カレンダー登録で通知を代替します。

対象：

- 全試合
- 日本戦
- メイン推し国
- サブ推し国
- 重要度S/A
- 今日見るべき試合

出力仕様：

- `.ics` ファイルを生成
- イベント名に対戦カードと重要度を入れる
- 説明欄に見る理由を入れる
- 通知タイミングはカレンダー側の設定に委ねる

### 8.9 X共有カード生成

Webアプリへの流入導線として、画像カードを生成します。

カード種類：

- 今日見るべき試合カード
- 日本/推し国の突破条件カード
- 3位通過ラインカード
- 寝不足危険度カード
- 推し国の次戦カード

MVP仕様：

- Web上で画像生成
- PNGダウンロード
- X投稿文はテキストでコピー可能
- 自動投稿はしない

### 8.10 データ自動更新

試合結果の手入力地獄を避けるため、無料APIとGitHub Actionsによる更新を検証します。

MVP仕様：

- 無料スポーツデータAPIから試合結果を取得
- GitHub Actionsで定期実行
- `results.json` を更新
- アプリ側はJSONを読み込んで順位再計算
- API不調時は手動補正用ファイルで上書き

## 9. 非機能要件

### 9.1 コスト

- 原則無料枠内で運用
- 有料API契約なし
- 有料ホスティングなし
- GitHub Pages公開を基本とする
- Replitは開発・動作確認環境として使う

### 9.2 パフォーマンス

- スマホ表示で軽いこと
- 初回表示3秒以内を目標
- JSONファイルは必要最小限にする
- 動画や重い画像は使わない

### 9.3 信頼性

- 順位計算はユニットテストを作る
- 3位通過ライン計算はテストケースを作る
- API取得失敗時は前回データを保持する
- 手動補正データを優先できる設計にする

### 9.4 保守性

- データとロジックを分離する
- UIコンポーネントを分割する
- 計算ロジックはTypeScript関数で独立管理する
- Codexに修正依頼しやすい構成にする

### 9.5 権利・表現リスク

- 公式ロゴ・公式エンブレム・公式画像は使わない
- 公式風デザインにしない
- 非公式ファン向けツールであることを明記する
- データ取得元の利用条件を確認する
- スクレイピングは本命にしない
- 必要最小限の試合データのみ扱う

## 10. AI利用方針

AIに任せるもの：

- 試合の見る理由
- 過去対戦・因縁の整理
- 試合前コメント
- 試合後コメント
- X投稿文案
- 共有カードのコピー
- UI文言改善

AIに任せないもの：

- 勝点計算
- 得失点計算
- グループ順位判定
- 3位通過ライン判定
- トーナメント分岐
- 試合結果の正誤判定

## 11. 成功条件

MVP時点の成功条件は以下です。

- スマホで快適に閲覧できる
- 日本と推し国を設定できる
- 今日見るべき試合がS/A/B/Cで表示される
- グループ順位表が自動計算される
- 3位通過ラインが可視化される
- 試合詳細で「なぜ見るべきか」がわかる
- X共有カードを生成できる
- ICSカレンダーを出力できる
- 試合結果取得をAPIまたは補正ファイルで更新できる

## 12. 優先順位

最初から完璧なプロダクトを作らず、MVPを優先します。

優先順位は以下です。

1. まずスマホWebアプリとして動かす
2. ダミーデータで順位計算と3位通過ラインを検証する
3. 推し国設定と今日見るべき試合ランキングを作る
4. XカードとICS出力を追加する
5. API結果取得とGitHub Actions自動更新を追加する

機能追加よりも、完成と運用可能性を優先します。
