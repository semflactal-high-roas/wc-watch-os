# 今日見るべきW杯 - 観戦OS開発

## 1. プロジェクト概要

このプロジェクトは、日本時間でW杯を見る国内ユーザー向けに、スマホで使える観戦判断ダッシュボードを作るためのものです。

単なる試合日程表ではなく、日本代表とユーザーが選んだ推し国に関係ある試合だけを、見る価値・突破条件・3位通過ライン・トーナメント分岐つきで提示します。

## 2. 一言価値提案

104試合、全部追わなくていい。  
日本と推し国に関係ある試合だけ教えます。

## 3. 解決したい課題

2026年大会は試合数が多く、グループ3位の上位チームも決勝トーナメントに進むため、ライト〜中級ファンにとって状況がわかりにくくなります。

このプロダクトでは、以下の疑問を解消します。

- 今日どの試合を見るべきか
- 深夜・早朝に起きる価値がある試合はどれか
- 日本代表や推し国の突破条件は何か
- 日本や推し国と直接対戦しない国の試合が、なぜ関係するのか
- 3位通過ラインがどう動いているのか
- このまま進むと決勝トーナメントでどこと当たりそうか
- 朝起きた時に、何を把握すれば追いつけるのか

## 4. 想定ユーザー

主な対象は、日本国内でW杯を観戦するライト〜中級ファンです。

特徴は以下です。

- 日本代表は見る
- 海外の強豪国や推し国も2〜3チーム気になる
- 全試合を追う余裕はない
- 試合時間が深夜・早朝の場合、見る価値を判断したい
- グループ突破条件や3位通過ラインは詳しくない
- SNSや職場・友人との会話には乗りたい
- 速報アプリではなく、試合の意味の整理が欲しい

## 5. プロダクト方針

基本思想は、情報を増やすことではなく、見るべき情報だけに減らすことです。

このプロダクトは、ライブスコアアプリではありません。  
スコアや順位をもとに、「次に何を見るべきか」「その試合がなぜ重要なのか」を判断しやすくする観戦OSです。

## 6. 主な機能

初期MVPでは、以下を実装対象とします。

- スマホ最適化Webアプリ
- 日本時間の全試合スケジュール
- 日本代表＋推し国設定
- 今日見るべき試合ランキング
- グループ順位表
- 3位通過ライン
- 試合詳細ページ
- 試合ごとの見る理由
- X共有カード生成
- ICSカレンダー出力
- 無料APIによる試合結果取得の基本実装

## 7. 初期対象外

初期開発では、以下は作りません。

- ネイティブアプリ
- LINE通知
- ログイン機能
- 課金機能
- リアルタイム速報
- 試合中ライブ更新
- 勝敗予想メイン機能
- コミュニティ機能
- コメント欄
- 自動X投稿
- 公式大会ロゴ・公式素材を使ったデザイン

## 8. 使用ツール

このプロジェクトは、可能な限り低コストで、以下のツールを使って進めます。

- ChatGPT：要件整理、仕様レビュー、解説文生成、テスト観点整理
- Codex：実装、リファクタリング、バグ修正、テスト作成
- Grok：X上の話題収集、投稿切り口の探索、サッカーファン向け言い回しの補助
- Replit：開発・動作確認環境
- GitHub：コード管理、Issue管理、GitHub Pages公開、GitHub Actions自動更新

## 9. 技術方針

初期構成は以下を想定します。

- フロントエンド：Vite + React + TypeScript
- スタイリング：Tailwind CSS
- データ管理：JSONファイル
- ユーザー設定：localStorage
- 可視化：HTML table / SVG / Canvas
- 共有カード生成：HTML Canvas
- カレンダー出力：ICSファイル生成
- 結果取得：無料スポーツデータAPIを優先
- 自動更新：GitHub Actions
- 公開：GitHub Pages

## 10. AI利用方針

AIに任せるもの。

- 試合の見る理由
- 過去対戦・因縁の整理
- 試合前コメント
- 試合後コメント
- X投稿文案
- 共有カードのコピー
- UI文言改善

AIに任せないもの。

- 勝点計算
- 得失点計算
- グループ順位判定
- 3位通過ライン判定
- トーナメント分岐
- 試合結果の正誤判定

順位計算や突破条件判定は、TypeScriptロジックで決定論的に処理します。

## 11. 成功条件

MVP時点の成功条件は以下です。

- スマホで快適に閲覧できる
- 日本と推し国を設定できる
- 今日見るべき試合がS/A/B/Cで表示される
- グループ順位表が自動計算される
- 3位通過ラインが可視化される
- 試合詳細で「なぜ見るべきか」がわかる
- X共有カードを生成できる
- ICSカレンダーを出力できる
- 試合結果取得をAPIまたは補正ファイルで更新できる

## 12. 開発方針

最初から完璧なプロダクトを作らず、MVPを優先します。

優先順位は以下です。

1. まずスマホWebアプリとして動かす
2. ダミーデータで順位計算と3位通過ラインを検証する
3. 推し国設定と今日見るべき試合ランキングを作る
4. XカードとICS出力を追加する
5. API結果取得とGitHub Actions自動更新を追加する

機能追加よりも、完成と運用可能性を優先します。
